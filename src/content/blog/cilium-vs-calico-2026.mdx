This is a practical comparison based on real production use, not vendor marketing.

## Quick Comparison

| Feature | Cilium | Calico |
|---------|--------|--------|
| Data plane | eBPF | iptables/eBPF/eBPF XDP |
| Network policy | L3/L4/L7 | L3/L4 (L7 via Envoy) |
| Encryption | WireGuard or IPsec | WireGuard or IPsec |
| Observability | Hubble (built-in) | Enterprise only |
| Service mesh | Built-in (no sidecar) | Calico + Envoy |
| Performance | High (bypasses iptables) | Good (eBPF mode) |
| Maturity | CNCF Graduated | Widely deployed |
| Bandwidth management | Yes | No |

## When to Use Cilium

- **L7 network policies**: HTTP method/path-aware policies without a sidecar
- **Observability**: Hubble provides real-time network flow visibility out of the box
- **Performance-critical clusters**: eBPF bypasses iptables entirely, reducing latency and CPU overhead
- **Service mesh without sidecars**: Cilium can replace Istio/Linkerd for basic service mesh features

```yaml
# Cilium L7 policy — allow only GET /api
apiVersion: cilium.io/v2
kind: CiliumNetworkPolicy
metadata:
  name: allow-api-reads
spec:
  endpointSelector:
    matchLabels:
      app: backend
  ingress:
    - fromEndpoints:
        - matchLabels:
            app: frontend
      toPorts:
        - ports:
            - port: "8080"
          rules:
            http:
              - method: GET
                path: "/api/.*"
```

## When to Use Calico

- **Established environments**: Calico has been the default CNI for many managed Kubernetes platforms
- **Hybrid networking**: Strong support for BGP peering with existing network infrastructure
- **Windows nodes**: Calico supports Windows containers natively
- **Existing investment**: If your team already knows Calico, the eBPF mode gives you performance parity

```yaml
# Calico GlobalNetworkPolicy
apiVersion: projectcalico.org/v3
kind: GlobalNetworkPolicy
metadata:
  name: deny-all-default
spec:
  selector: all()
  types:
    - Ingress
    - Egress
  ingress:
    - action: Deny
  egress:
    - action: Deny
```

## Performance

In benchmark tests with large clusters (1000+ pods):
- **Cilium** shows lower latency due to eBPF-native packet processing
- **Calico eBPF mode** closes the gap significantly vs iptables mode
- Both handle 10 Gbps+ throughput in production

## My Recommendation

Choose **Cilium** for greenfield clusters, especially if you want built-in observability (Hubble) and L7 policies without sidecars. Choose **Calico** if you need BGP peering, Windows node support, or have an existing Calico deployment. Both are excellent.
