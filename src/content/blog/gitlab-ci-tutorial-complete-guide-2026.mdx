CI/CD pipelines are the backbone of modern software delivery. Here is how to set them up properly.

## Step-by-Step Guide

### Define Your Pipeline

A solid CI/CD pipeline includes:

1. **Build** — compile, install dependencies
2. **Test** — unit tests, integration tests, linting
3. **Scan** — vulnerability scanning, SAST, SCA
4. **Deploy** — staging first, then production with approval

### Example Pipeline

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run tests
        run: |
          npm ci
          npm test
          npm run lint

  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Trivy scan
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: fs
          severity: CRITICAL,HIGH

  deploy:
    needs: [test, security]
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - name: Deploy to production
        run: echo "Deploying..."
```
