This is a practical comparison based on real production use, not vendor marketing.

## Quick Comparison

| Feature | K3s | K8s (kubeadm) |
|---------|-----|---------------|
| Binary size | ~70 MB | Multiple components (~500 MB+) |
| Memory (control plane) | ~512 MB | ~2 GB+ |
| Install time | 30 seconds | 15-30 minutes |
| Default datastore | SQLite/etcd | etcd |
| Certified Kubernetes | Yes (CNCF conformant) | Yes |
| HA support | Yes (embedded etcd) | Yes |
| ARM support | Native | Requires manual setup |
| Components | Bundled (containerd, Flannel, CoreDNS, Traefik) | Choose your own |

## When to Use K3s

- **Edge and IoT**: Runs on Raspberry Pi, ARM devices, and minimal hardware
- **Development and CI/CD**: Instant single-node clusters for testing
- **Small production clusters**: 1-20 nodes where simplicity matters
- **Learning Kubernetes**: Full Kubernetes API without the setup complexity

```bash
# Install K3s — literally one command
curl -sfL https://get.k3s.io | sh -

# Check cluster
sudo k3s kubectl get nodes

# Add a worker
curl -sfL https://get.k3s.io | K3S_URL=https://master:6443   K3S_TOKEN=$(sudo cat /var/lib/rancher/k3s/server/node-token) sh -
```

## When to Use K8s (kubeadm)

- **Full control**: Choose every component — CNI, CSI, ingress controller, runtime
- **Large clusters**: 100+ nodes with advanced networking and storage requirements
- **Enterprise environments**: When you need specific CNI (Cilium, Calico), storage (Ceph, Portworx), and observability
- **Managed Kubernetes base**: kubeadm is what EKS, AKS, and GKE build upon

## What K3s Removes

K3s is Kubernetes with the following replaced or removed:
- **etcd** → SQLite (single node) or embedded etcd (HA)
- **Cloud controller manager** → removed (no cloud provider integration)
- **Storage plugins** → in-tree plugins removed, CSI supported
- **Bundled**: Flannel CNI, CoreDNS, Traefik ingress, local-path storage

You can replace any bundled component — for example, install Cilium instead of Flannel.

## My Recommendation

Start with **K3s** for development, edge, and small production workloads. Use **kubeadm** (or a managed service like EKS/GKE/AKS) when you need large-scale production clusters with full component control. K3s is real Kubernetes — same API, same kubectl, same Helm charts.
