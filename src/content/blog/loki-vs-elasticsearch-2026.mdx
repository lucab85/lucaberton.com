This is a practical comparison based on real production use, not vendor marketing.

## Quick Comparison

| Feature | Loki | Elasticsearch |
|---------|------|---------------|
| Indexing | Labels only (like Prometheus) | Full-text indexing |
| Storage cost | Low (compressed chunks in object storage) | High (inverted index) |
| Query language | LogQL | KQL / Lucene |
| Resource usage | Low | High (JVM heap) |
| Full-text search | Grep-like (brute force on chunks) | Native (inverted index) |
| Integration | Grafana (native) | Kibana |
| Multi-tenancy | Built-in | X-Pack (paid) |
| Scalability | Horizontal (microservices) | Horizontal (shards) |

## When to Use Loki

- **Cost-sensitive environments**: Loki stores only labels in the index — log content goes to cheap object storage (S3, GCS)
- **Grafana stack**: If you already use Prometheus + Grafana, Loki is the natural logging choice — same label model, same dashboards
- **Kubernetes-native**: Promtail or Grafana Agent collects logs with Kubernetes labels automatically
- **Simple queries**: Most log queries are "show me logs for this service in the last hour" — Loki handles this efficiently

```
# LogQL — same label model as PromQL
{namespace="production", app="api"} |= "error" | json | status >= 500
```

## When to Use Elasticsearch

- **Full-text search**: If you need to search across millions of log lines by arbitrary text patterns
- **Complex analytics**: Elasticsearch aggregations, visualizations, and machine learning features
- **APM and SIEM**: Elastic APM and Security (SIEM) are mature products built on Elasticsearch
- **Non-Kubernetes environments**: Better tooling for VMs, bare metal, and legacy systems

## Cost Comparison

For a cluster ingesting 100 GB/day of logs:
- **Elasticsearch**: ~3 hot data nodes (16 GB RAM each) + object storage = **$800-1200/mo**
- **Loki**: ~2 small instances + S3/GCS storage = **$200-400/mo**

Loki is 3-5x cheaper because it does not build an inverted index for log content.

## My Recommendation

Use **Loki** for Kubernetes environments where cost matters and you already use Grafana. Use **Elasticsearch** when you need full-text search, APM, or SIEM capabilities.
