## What You'll Learn

- ✅ Explain the roles of metrics, logs, and traces and map Prometheus, Grafana, and OpenTelemetry to each signal
- ✅ Deploy a minimal local stack: scrape metrics with Prometheus, route telemetry via OTel Collector, and visualize in Grafana
- ✅ Instrument a sample app with OpenTelemetry and confirm traces/metrics flow end-to-end
- ✅ Build a practical three-panel Grafana dashboard tracking requests, errors, and latency
- ✅ Configure noise-aware alerts with proper thresholds and time windows
- ✅ Navigate from a dashboard panel to related spans using exemplars for root-cause analysis

## Course Overview

Modern cloud-native applications rarely crash outright. Instead, they fail in subtle ways — latency spikes, partial errors, or noisy dependencies. This course helps you become productive with the open-source observability trio and teaches key patterns like pull versus push collection, label hygiene, histogram quantiles, and Collector pipelines.

## Key Observability Patterns Covered

Throughout the course, you will learn essential observability patterns that apply to any production environment:

- **Pull vs. Push collection** — understanding when to use each approach
- **Label hygiene** — keeping cardinality under control for sustainable metrics
- **Histogram quantiles** — computing meaningful latency percentiles with `histogram_quantile()`
- **Collector pipelines** — configuring receivers → processors → exporters
- **Exemplars** — correlating metrics with traces for rapid root-cause analysis
- **Dashboard JSON export** — version-controlling your dashboards
