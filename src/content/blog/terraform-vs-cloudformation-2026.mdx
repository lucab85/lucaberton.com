This is a practical comparison based on real production use, not vendor marketing.

## Quick Comparison

| Feature | Terraform | CloudFormation |
|---------|-----------|----------------|
| Cloud support | Multi-cloud (AWS, Azure, GCP, etc.) | AWS only |
| Language | HCL (declarative) | JSON/YAML |
| State management | State file (remote backend) | Managed by AWS |
| Drift detection | `terraform plan` | Drift detection feature |
| Module ecosystem | Terraform Registry (10K+ modules) | CloudFormation Registry |
| Preview changes | `terraform plan` | Change sets |
| Rollback | Manual (apply previous state) | Automatic stack rollback |
| Cost | Free (BSL license) / Terraform Cloud | Free (AWS service) |

## When to Use Terraform

- **Multi-cloud or hybrid**: Managing AWS + Azure + GCP + Kubernetes from one tool
- **Team collaboration**: Terraform Cloud/Enterprise provides state locking, policy-as-code, run approvals
- **Ecosystem**: HCL is more readable than JSON/YAML for infrastructure, and the provider ecosystem is massive
- **Portability**: If you might change cloud providers, Terraform abstractions help

```hcl
# Terraform — multi-cloud in one codebase
resource "aws_instance" "web" {
  ami           = "ami-0abcdef12345"
  instance_type = "t3.medium"
}

resource "azurerm_kubernetes_cluster" "aks" {
  name                = "my-aks"
  location            = "westeurope"
  dns_prefix          = "myaks"
}
```

## When to Use CloudFormation

- **AWS-only shops**: If you are 100% AWS, CloudFormation integrates natively with every AWS service on day one
- **Automatic rollback**: Failed deployments automatically roll back to the previous working state
- **No state file management**: AWS manages state for you — no S3 backend, no DynamoDB lock table
- **AWS CDK**: Write CloudFormation in TypeScript, Python, Java, or Go using AWS CDK

```yaml
# CloudFormation
Resources:
  WebServer:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: ami-0abcdef12345
      InstanceType: t3.medium
```

## The OpenTofu Factor

Since Terraform moved to the BSL license in 2023, **OpenTofu** (Linux Foundation fork) provides a truly open-source alternative with full Terraform compatibility. If licensing matters to your organization, evaluate OpenTofu.

## My Recommendation

Use **Terraform** (or OpenTofu) for multi-cloud, hybrid, or Kubernetes-centric infrastructure. Use **CloudFormation** (especially with CDK) if you are all-in on AWS and want zero state management overhead.
