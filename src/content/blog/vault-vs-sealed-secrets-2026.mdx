Vault is an enterprise secrets management platform. Sealed Secrets is a Kubernetes controller that encrypts secrets for Git. They solve different problems at different scales.

## Architecture

| Aspect | Vault | Sealed Secrets |
|---|---|---|
| **Type** | Full secrets platform | Kubernetes controller |
| **Scope** | Multi-platform (K8s, VMs, apps) | Kubernetes only |
| **Dynamic secrets** | Yes (database, cloud, PKI) | No (static only) |
| **Encryption** | Transit engine, auto-unseal | Asymmetric encryption (public/private key) |
| **Access control** | Policies (fine-grained) | Kubernetes RBAC |
| **Audit** | Full audit log | Kubernetes audit log |
| **HA** | Raft, Consul | Controller replica |
| **License** | BSL 1.1 (community), Enterprise | Apache 2.0 |

## How they work

### Sealed Secrets

```bash
# Install controller
helm install sealed-secrets sealed-secrets/sealed-secrets

# Encrypt a secret (client-side)
echo -n "mypassword" | kubectl create secret generic db-creds \
  --from-literal=password=$(cat -) --dry-run=client -o yaml | \
  kubeseal --format yaml > sealed-secret.yaml
```

```yaml
# sealed-secret.yaml — safe to commit to Git
apiVersion: bitnami.com/v1alpha1
kind: SealedSecret
metadata:
  name: db-creds
spec:
  encryptedData:
    password: AgBy3i4OJSWK+PiTySYZZA9rO... # encrypted with controller's public key
```

The SealedSecret is encrypted with the controller's public key. Only the controller (running in your cluster) can decrypt it. Safe to store in Git.

### Vault

```bash
# Store a secret
vault kv put secret/db/creds username=admin password=s3cret

# Application retrieves secret via API
vault kv get -field=password secret/db/creds

# Dynamic database credentials (auto-rotating)
vault write database/roles/readonly \
  db_name=mydb \
  creation_statements="CREATE ROLE \"{{name}}\" WITH LOGIN PASSWORD '{{password}}' VALID UNTIL '{{expiration}}';" \
  default_ttl="1h" \
  max_ttl="24h"

# Each request gets unique, time-limited credentials
vault read database/creds/readonly
# Key        Value
# username   v-root-readonly-abc123
# password   A1aBcDeFgHiJkL
# ttl        1h
```

## GitOps compatibility

### Sealed Secrets + GitOps

```yaml
# ArgoCD deploys SealedSecrets from Git — works perfectly
# secrets/production/sealed-db-creds.yaml
apiVersion: bitnami.com/v1alpha1
kind: SealedSecret
metadata:
  name: db-creds
  namespace: production
spec:
  encryptedData:
    password: AgBy3i4OJSWK+PiTySYZZA9rO...
```

Sealed Secrets are designed for GitOps — encrypted secrets live in Git alongside application manifests.

### Vault + GitOps

```yaml
# Vault Secrets Operator — syncs Vault secrets to Kubernetes
apiVersion: secrets.hashicorp.com/v1beta1
kind: VaultStaticSecret
metadata:
  name: db-creds
spec:
  type: kv-v2
  mount: secret
  path: db/creds
  destination:
    name: db-creds
    create: true
  refreshAfter: 30s
```

Vault works with GitOps via the Vault Secrets Operator or External Secrets Operator — Git stores the reference, not the secret itself.

## Feature comparison

| Feature | Vault | Sealed Secrets |
|---|---|---|
| **Dynamic secrets** | Yes (database, AWS, Azure, GCP) | No |
| **Secret rotation** | Automatic (TTL-based) | Manual (re-encrypt and commit) |
| **PKI/certificates** | Built-in CA | No |
| **Encryption as a service** | Transit engine | No |
| **Multi-platform** | K8s, VMs, Lambda, CI/CD | Kubernetes only |
| **Setup complexity** | High (Vault cluster, unseal, policies) | Low (one Helm install) |
| **Operations** | Complex (backups, HA, upgrades) | Simple (single controller) |
| **Cost** | Free (community) or $$$$ (Enterprise) | Free |
| **Secret sprawl** | Centralized (single source of truth) | Per-cluster encrypted secrets |

## Decision guide

### Choose Sealed Secrets when:
- **GitOps** — you want secrets in Git alongside manifests
- **Simple Kubernetes secrets** — static passwords, API keys, tokens
- **Small teams** — no dedicated security/platform team
- **Quick setup** — working in 5 minutes
- Secret rotation is **infrequent** (monthly/quarterly manual rotation is acceptable)

### Choose Vault when:
- **Dynamic secrets** — auto-generated, time-limited database/cloud credentials
- **Multi-platform** — secrets for Kubernetes + VMs + CI/CD + applications
- **Compliance** — audit trail, fine-grained policies, encryption as a service
- **Secret rotation** — automatic, TTL-based rotation
- **PKI** — issue and manage TLS certificates
- **Enterprise** — Vault Enterprise with namespaces, replication, HSM support

### Also consider
- **External Secrets Operator** — sync secrets from AWS Secrets Manager, Azure Key Vault, GCP Secret Manager, or Vault into Kubernetes Secrets
- **SOPS** — encrypt YAML/JSON files with age/PGP/cloud KMS — another GitOps-friendly option
